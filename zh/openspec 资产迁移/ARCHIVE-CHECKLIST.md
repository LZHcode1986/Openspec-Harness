# 归档后主规范正规化检查清单

执行 `archive` 后，必须立刻进行主规范正规化整理，避免归档结果污染主规范库。

## 1. 结构完整性
- [ ] 归档后的 `spec.md` 是否仍保留正确的标题？
- [ ] 是否有独立的 `## Purpose` 章节？
- [ ] 是否有独立的 `## Requirements` 章节？

## 2. 消除增量语义
- [ ] 主 spec 中是否仍包含 `## ADDED Requirements`、`## MODIFIED Requirements` 等增量标题？
- [ ] 需求描述中是否仍包含“本次修改了...”、“系统现在需要...”等增量措辞？

## 3. 消除占位符
- [ ] `Purpose` 是否仍是 `TBD`、归档占位文案或草稿式表述？
- [ ] `Requirements` 是否仍保留未替换的模板占位符？

## 4. 可独立阅读性
- [ ] 合并后的主 spec 是否能在不参考已归档 change 的前提下独立表达当前系统行为？

## 5. 校验工具验证
- [ ] 运行项目对应的 `openspec validate --all --strict` 或等效校验命令，确认全部通过？
